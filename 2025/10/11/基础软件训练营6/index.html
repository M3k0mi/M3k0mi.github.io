# linux 目录

# afs

空目录，作为临时文件的临时挂载使用

# bash_completion.d

系统自动生成

# bin

存放操作系统所有的命令，原路径 /usr/bin

# boot

操作系统启动引导分区目录，系统开机时加载的文件

# dev

操作系统各种各样的设备文件，比如各种外接设备文件、分区文件，删除后重启依旧生成

# etc

各种配置文件，系统配置和软件配置，比如主机名、网卡配置文件、防火墙配置、用户账号密码

# home

存放普通用户的 home 目录，root 单独一个目录

# lib

命令会引用的库文件

# lost+found

存储电脑意外关机的数据

# media/mnt

临时挂载使用，比如要使用光盘的文件，必须将其挂载

挂载到 media

# opt

存放的都是用户下载的第三方文件的保存路径

# proc

存放系统运行的数据，还有系统信息，文件名称都是进程 id，重启后会清空

# root

超级管理员的 home 目录

# run

保存系统开机时的运行数据

# sbin

保存二进制可执行文件 -> 命令，只有超级管理员可以使用

# srv

空目录，一些变化的数据存储目录，比如 apache 的网页文件

# sys

system，存储系统的相关配置，系统调优可能会修改

# tmp

软件运行时的临时文件

# usr

最重要，快捷方式的源文件路径
/usr/local，用户安装第三方软件的路径

# var

程序的默认日志文件、网页文件

# 文件规范命名

1. 字符不要超过 255
2. 文件名字中不要包含特殊字符

# 文件路径

分为相对路径和绝对路径

相对路径：只有特定的目录下才能使用
绝对路径：在系统上所有的位置都能使用

# linux 命令

# cd

进入某个文件夹、目录

进入根目录

回到上次所在的目录

快速回到当前用户所在的家目录

快速回到某一用户的家目录

回到上一级目录

# ls

列出文件或目录

列出当前，目录下的目录或文件

列出根目录下的内容

查看根目录内容及详细信息

文件类型

- 普通文件
d 目录文件
b 块文件
c 字符设备 外接设备 键盘鼠标
l 链接文件，符号链接，快捷方式
s 套接字文件，系统程序之间的通信可以通过套接字文件通信

文件权限

r 可读
w 可写
x 可执行

三个分别为人、组、其他人的权限

将文件的大小进行换算

显示隐藏文件，. 开头的为隐藏文件

每个目录下都有。和..，. 为当前目录，.. 为上一级目录

列出目录本身

# pwd

打印当前，目录

# passwd

修改用户密码

管理员 -> 修改任何用户密码，不需要验证密码，不用遵循密码策略

普通用户 -> 只能修改自己的密码，且需要验证密码，且要遵循密码策略

# cat

打印文件内容

并排序

# touch/mkdir

创建文件 / 目录

touch：创建空文件、更新文件 / 目录的时间戳

递归创建，将没有的目录一并创建

# cp

拷贝文件及目录

拷贝一整个目录，. 处可以改，表示拷贝并重命名

# mv

移动和重命名

# rm

删除文件

rm 删除目录要将文件全部删除

将目录下的文件一并删完

# vim

文本编辑器

查看文件、编辑文件、保存文件

# vim 的三大模式

命令模式：可以实现简单的复制、粘贴、删除内容以及跳转

编辑模式 /’插入模式：编辑文本的内容，增删改

退出模式 / 末行模式：保存文件、另存为文件

三大模式之间的切换：默认打开的是默认模式
编辑模式和退出模式无法直接切换，中间必须要经过命令模式
编辑模式 -> 命令模式 -> 退出模式
退出模式 -> 命令模式 -> 编辑模式

# vim 快捷方式

快捷键 描述
i 在光标所在处切换到编辑模式
a 在光标后面一个字符切换到编辑模式
o 在光标所在行的下一行切换到编辑模式
I 在光标所在行的最前面切换到编辑模式
A 在光标所在行的最后面切换到编辑模式
O 在光标所在行的最前面切换到编辑模式

从命令模式切换到退出模式：':'

从退出模式 / 插入模式切换到命令模式：‘ESC’

快捷键 描述
yy 复制当前光标所在的行
p 在光标下面一行进行粘贴
P 在光标上面一行进行粘贴
dd 删除光标所在行一整行内容
dG 删除光标所在行以及后面所有的内容
nG (n 表示数字) 跳转到第 n 行内容
G 跳转到文件的最后一行
gg 跳转到文件的第一行
x 删除光标所在处单个字符
u 撤销上一次的操作
退出模式下常见的快捷方式 描述
:set number 显示文件每一行的行号
:n (n 是数字) 直接跳转到第 n 行
:w 文件路径 将当前文件另存为
:%d 清空当前文件的所有内容
😒/ 关键字 / 替换后的关键字 /g 替换当前光标所在行的所有关键字 (默认不加 g 则是替换第一个匹配到的关键字)
:$s / 关键字 / 替换后的关键字 /g 替换文件中所有的关键字

在命令模式下：输入 '/ 关键字 ' 可以匹配关键字，通过小写 n 来进行下一个关键字的跳转。大写的 N 表示跳转到上一个关键字

# 用户管理和权限管理

# 用户管理

用户的作用：

1. 通过用户登录系统获得操作环境
2. 用户可以来运行软件，所有的软件都需要用户来运行

系统根据用户 id 来识别的
管理员是根据用户名来识别用户的

系统超级管理员：UID 为 0 的用户就是系统的超级管理员
系统用户：0<UID<1000, 这些用户一般不可以登录到操作系统，用来专门运行各种软件和程序
普通用户:>=1000UID，这些用户都是超级管理员创建

用户组：多个用户的集合，每个用户必须要属于某一个组

分类：
1. 私有组 (属组)：用户自己的用户组，创建用户的同时也会创建一个同名的私有组
2. 附加组 (公共组)：用户可以加入到其他组中，目的是为了获得对应组的权限

# /etc/passwd 用户信息文件

用户名和 UID 的对应关系保存在该文件中

root:x:0:0:root:/root:/bin/bash

第一列：用户名
第二列：密码占位符，x 表示这个用户登录时需要密码验证，没有 x 则表示不需要
第三列：表示用户的 id
第四列：表示拥有组的 id
第五列：用户的描述信息，没有实际意义
第六列：用户的家目录
第七列：登录用户第一个执行的程序，/bin/bash 表示用户可以登录，可以获取到对应的执行环境，/sbin/nologin 表示用户不可以登录，系统用户大多数都是这个登录 shell
