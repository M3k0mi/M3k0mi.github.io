# 磁盘管理、文件系统、逻辑卷

分区的好处

1. 每个分区都可以创建不同的文件系统
2. 每个分区是独立的，分区故障不影响其他分区
3. 分区可以实现文件的区分，不同的分区保存的内容不一样，方便管理

linux 必须要进行分区
1./boot 分区，存储系统开机引导的文件
2./ 分区

分区方案
MBR 方案
特点：
1. 如果磁盘单个分区超过 2TB，BMR 分区方案不支持
2.MBR 方案有分区类型区分，有主分区、扩展分区、逻辑分区 (在扩展分区内划分的)，最多支持 3 给主分区 1 个扩展分区

GPT 方案
特点：
1. 磁盘单个分区可以超过 2TB
2.GPT 分区方案实际上没有限制数量，128 个主分区 (实际上是操作系统的限制)

默认都是 GPT 分区方案

# 分区工具

parted: 支持非交互式分区，支持 MBR 和 GPT 分区方案，支持在脚本中使用 parted 来对磁盘分分区

fdisk

gdisk: 专门对 GPT 分区方案进行分区

parted 交互式分区

# scsi/sata 硬盘

/dev/sda 第一块 scsi 硬盘
/dev/sda1 第一块 scsi 硬盘的第一个分区
/dev/sda2 第一块 scsi 硬盘的第二个分区
/dev/sdb 第二块 scsi 硬盘
/dev/sdc 第三块 scsi 硬盘

# fdisk

# 文件系统

文件系统是文件在硬盘上的存储方式和组织结构 --> 文件是存储到文件系统上的，文件系统是在磁盘上创建的

windows 主流文件系统 NTFS
linux 主流文件系统 xfs,ext

创建文件系统 --> 格式化

新的文件系统会覆盖旧的文件系统导致文件丢失 (格式化就是在创建文件系统)

# 挂载 (linux 系统独有概念)

将文件系统挂载给某个目录，这个目录就称之为挂载点
往挂载点中写数据，实际上就是先写入到对应的文件系统中

# mount

将 /dev/sdb1 挂载到 /data1 目录

卸载

临时挂载，重启系统会失效，但是不会丢失文件

永久挂载需要写入 /etc/fstab 文件

blkid

查看文件系统 UUID
