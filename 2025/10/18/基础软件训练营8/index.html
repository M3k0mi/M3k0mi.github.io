# 三大特殊权限

# suid

对于文件：如果一个文件拥有 suid 权限，那么再执行这个文件的时候会以文件拥有人的身份执行，只有二进制可执行文件有效果，脚本文件没有任何效果

对目录：目录赋予 suid 权限没有任何意义

ll -d 目录

查看是否有 suid 权限，拥有人栏位 x 变成 s 则拥有，拥有人对文件没有 x 权限，则为大写 S

赋予 suid 权限

删除 suid 权限

# sgid

对于文件：如果一个文件拥有 sgid 权限，再执行这个文件的时候会以文件的拥有组身份去执行

对于目录：一个目录如果具有 sgid 权限，则这个目录下所有文件都会继承这个目录的拥有组

赋予目录 sgid 权限

# sticky(sbit)

粘性权限
对于文件：没有意义
对于目录：如果一个目录拥有 sbit 权限，则这个目录，自己创建的文件才能修改和重命名

其他的的 x 权限变成 t

赋予 sbit 权限

# 软件包管理和服务管理

openEuler 系统，基于 RHEL 的发行版本
基于 debian 的发行版本

软件包管理方式不一样

RHEL: 基于 rpm 包进行管理

openEuler 系统，软件包格式是 rpm 包，软甲包的后缀通常来说是 rpm

rpm:redhat package management 红帽包管理器

# 获取 rpm 包途径

1.openEuler 官网
2. 上国内开源镜像站

# rpm 管理工具

1. 安装、卸载和重装

-i install
-v 显示详细信息
- h 显示进度条

删除软件

查询

升级软件包

-U 如果软件存在则升级，如果不存在则安装
- F 如果软件存在则升级，如果不存在则报错
升级软件包，并且可以在线升级

重新安装软件包

--reinstall

2. 查询

选项 描述
-q <包名> 查询软件是否安装
-qa 列出当前系统上所有已经安装的软件
-qi 查询软件包的信息 (包括源包是什么，在哪里可以下载，在哪里可以查询帮助以及软件包简单的描述)
-ql <包名> 列出软件包会释放的文件
-qc <包名> 列出软件包释放的配置文件
-qd <包名> 列出软件包的帮助文档
--changelog 查看软件的变更日志
-qf 查询文件的来源软件包

# 配置 yum 仓库步骤

# 本地仓库

存放到 iso 镜像文件

列出系统中所有可用的块设备信息

sr0 其实就是 /dev/sr0 设备，就是添加的光驱设备

1. 将光驱挂载给目录，作为光驱的访问入口

2. 修改 yum 配置文件，指向 yum 仓库

3. 执行 yum 命令测试

如果 yum 仓库配置有问题，.repo 配置文件有错误，yum makecache 会失败

一个 yum 的配置文件中可以有多个仓库

命令选项 描述
yum install 安装，依赖也安装
yum remove 卸载，依赖也卸载
yum reinstall 重装
yum update 更新
yum list 列出仓库中所有包
yum repolist 列出启用仓库
yum repolist all 列出所有仓库
yum info 查询软件包的信息
yum history 查看 yum 的历史
yum history undo 撤销某一次历史记录
yum grouplist 列出软件包组
yum groupinstall 安装软件包组
yum groupupdate 更新软件包组
yum group info 查看软件包组信息

dnf 和 yum 一样

# 部署 yum 仓库

主机名 网络地址 角色
yum_computer 192.168.2.146 yum 服务器
yum_client 192.168.2.147 内部节点
# yum_computer 操作

1. 安装 httpd 软件，前提是配置好了本地仓库

2. 关闭防火墙

3. 开启 httpd 服务

4. 在 /var/www/html/ 目录下创建一个 iso 目录

5. 将 nfs-utlis 的 rpm 包下载到上面的目录

6. 安装 createrepo 工具，目的是为了建立包和包的依赖关系表

# yum_client 操作

1. 备份 /etc/yum.repos.d 目录下的文件

2. 修改 yum 配置文件

3. 安装 nfs-utlis 软甲包
