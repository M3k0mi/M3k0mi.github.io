# /etc/shadow

用户密码文件

第一行：用户名
第二行：加密密码，用户密码必须是加密的，*、！、！！都表示没有密码，666、555 等都表示加密算法
第三行：表示上一次修改密码的日期，单位天，从 1970-1-1 开始计算 (Unix 系统诞生)
第四行：修改密码最短期限，上一次修改到下一次修改密码的间隔，必须经过... 天才能修改
第五行：密码最长有效期，密码有效的最长时间
第六行：密码过期前的告警天数
第七行：密码的过期后还有几天能使用
第八行：账户的失效日期，单位天
第九行：空

# 管理用户的命令

useradd 创建用户，-u 指定 uid，-g 指定 gid (必须有此 id 的组存在)，-c 指定用户描述信息，-d 指定用户的家目录，-s 指定登录 shell，-M 不要创建用户家目录，-O 允许使用重复的 uid 创建用户

userdel 删除用户，仅仅删除使用户，用户对应的家目录和邮箱文件都没有删除，此时此刻，如果创建新的用户 id 和删除用户的 id 一致，新用户就会继承

完整删除用户

usermod 修改用户，-l 修改用户名，-u 修改 uid，-g 修改 gid，-c 修改描述信息，-d 修改用户家目录 (不会创建目录)，-m 创建用户家目录 (通常 - md 联合使用)，-L 锁定用户密码 (实际修改了 /etc/shadow 文件中加密后的密码添加了一个！)，-U 解锁，-aG 将用户添加到某个组中

id 查询用户是否存在

创建用户则查找文件 uid 最大的用户 + 1 做为新用户的 id

# 目录权限

r 可以列出目录下的内容
w 表示可以创建和删除目录下的文件
x 可以进入目录。rx 联合使用，目录才有意义，表示可以列出目录下的内容。

# ACL 权限

访问控制列表，在基础权限之上更加精细化的管理

ACL 权限可以限定某一个用户对于一个文件的访问

如果一个文件有，ACL 权限优先级变化：
1. 如果文件有用户 acl 权限，则匹配用户 acl 权限
2. 匹配文件的原拥有人的权限
3. 匹配文件的拥有户组的 acl 权限
4. 匹配文件的原拥有组的权限
