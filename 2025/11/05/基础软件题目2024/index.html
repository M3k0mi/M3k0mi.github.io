# LVS 三种模式对比：

模式 工作原理 性能 网络要求 配置复杂度
NAT 修改 IP 地址 较低 同一网络 简单
DR 直接路由 最高 同一网络 中等
TUN IP 隧道 较高 可跨网络 复杂

# 实际应用场景：

  • NAT 模式：适合小规模应用，配置简单
  • DR 模式：适合高性能要求的场景
  • TUN 模式：适合真实服务器分布在不同机房的场景

在 openGauss 数据库系统中，CM（集群管理模块）主要负责集群的管理和监控功能，包括节点状态监控、故障检测、负载均衡和资源管理等。然而，CM 并不直接负责执行客户端发送的 SQL 命令或处理数据的存储和查询任务。这些功能是由数据库实例中的其他组件（如 CN（协调节点）和 DN（数据节点））处理的。CN 负责接收和解析 SQL 请求，协调查询执行，而 DN 负责实际的数据存储和查询处理。

编译型语言（如 C、C++）开发的程序在编译时生成的是特定于处理器架构的机器码。鲲鹏处理器基于 ARM 架构，而常见的 x86 程序无法直接在 ARM 架构上运行。因此，迁移到鲲鹏处理器时，通常需要重新编译源代码以生成 ARM 架构的二进制文件。如果程序依赖于特定平台的编译型语言库（如.so 库），这些库也需要重新移植和编译以兼容 ARM 架构。
